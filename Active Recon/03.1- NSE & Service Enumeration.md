## 🧠 **#4 Nmap: NSE, OS & Service Enumeration (Enhanced)**

---
###  **1. Discover Active Hosts (Network Sweeping)**

####  Method 1: Using Nmap (Ping Scan)

```bash
nmap -sn 192.168.1.1-254 -v | grep -v "host down"
```

- `-sn`: Skip port scan (ping only)
    
- `-v`: Verbose output
    
- `grep -v "host down"`: Filters out non-responding hosts
    

####  Method 2: Using Netdiscover (ARP Scan)

```bash
netdiscover -r 192.168.1.0/24
```

- ARP-based scanning — fast and silent on local networks
    
- **Best for discovering live hosts in LAN environments**
    

---

###  **2. OS Fingerprinting**

```bash
nmap -O 192.168.1.2
```

- `-O`: Enables **Operating System detection**
    
- Helps identify OS types and versions (useful for exploit selection)
    
- Requires root privileges
    

 **Note:** OS detection uses TCP/IP stack fingerprinting, which can be affected by firewalls or proxies.

---

###  **3. Service & Version Detection**

```bash
nmap -sV 192.168.1.24
```

- `-sV`: Detects **service versions** running on open ports
    
- Useful for identifying **vulnerable software versions**
    
- Combine with `-p-` for full port scan
    

 Example:

```bash
nmap -sV -p- 192.168.1.24
```

---

###  **4. Manual Verification of Services**

####  Netcat (nc) Verbose Connection

```bash
nc -nvv 192.168.1.2 21
```

- `-n`: No DNS resolution
    
- `-vv`: Very verbose (shows banner and connection steps)
    
- Helpful to manually grab banners for services like FTP, SSH, HTTP
    

####  Telnet (Banner Grabbing)

```bash
telnet 192.168.1.2 80
```

- Connect to a port and type basic HTTP request like `GET / HTTP/1.0`
    

####  Curl (Web Services)

```bash
curl -I http://192.168.1.2
```

- Retrieves HTTP headers — good for seeing web server version

###  **5. Vulnerability Research After Enumeration**

Once you know the **OS** or **service version**, search for exploits using:

-  **Google Hacking Database (GHDB)**  
    Example: `intitle:"index of" vsftpd 2.3.4`
    
- 💣 **Exploit DB**
    
    ```bash
    searchsploit vsftpd 2.3.4
    ```
    
-  **CVE Databases or GitHub PoCs**
    

---

###  **6. Nmap Scripting Engine (NSE)**

####  Script Location:

```bash
cd /usr/share/nmap/scripts/
```

#### ⚔️ Example NSE Script Usage:

```bash
nmap --script=dns-zone-transfer.nse -p 53 ns2.megacorpone.com
```

- Performs DNS zone transfer attack simulation
    

####  Most Useful NSE Scan Combo (Top 100 Ports + Scripts + Service Version):

```bash
nmap -sC -sV --top-ports=100 192.168.1.2
```

- `-sC`: Uses **default scripts** (equivalent to `--script=default`)
    
- `-sV`: Enables **version detection**
    
- `--top-ports=100`: Scan the **top 100 most common ports**
    
-  Preferred method because it gives **service info + basic vulnerabilities**
    

---

###  **Useful NSE Script Categories**

|Category|Example Script|Description|
|---|---|---|
|**Vuln Scan**|`vulners.nse`, `http-vuln-*`|Checks for known vulnerabilities|
|**Brute Force**|`ftp-brute.nse`, `ssh-brute`|Performs brute force login attempts|
|**Info Gather**|`http-title`, `http-headers`|Web server info and titles|
|**Exploit**|`smb-vuln-ms17-010`|Tests EternalBlue vulnerability|

---

###  **Saving Output for Reports**

```bash
nmap -sC -sV 192.168.1.2 -oN output.txt
```

- `-oN`: Normal output  
    Other formats:
    
- `-oX output.xml` (XML)
    
- `-oG output.gnmap` (grepable)
    

---

###  **Final Tips**

- Run as `root` for full scanning capabilities.
    
- Combine flags for **maximum efficiency**:
    
    ```bash
    sudo nmap -A -T4 192.168.1.2
    ```
    
    - `-A`: Aggressive (OS detection + version + script + traceroute)
        
    - `-T4`: Faster timing (less stealthy)
