## 👥 Lesson 3: User Info Gathering (People Recon)

---
### 🎯 1. Objective:

Gather **information about individuals working in the target organization**. This helps in:

- Crafting **social engineering** attacks (phishing, pretexting)
    
- Identifying **email patterns**
    
- Understanding internal roles, responsibilities, and structure
    
- Finding **public data leaks**
    

---

### 🔍 2. Where to Look?

#### 🧑‍💼 **LinkedIn**:

- Most valuable platform for corporate employee data.
    
- Search for employees by filtering with company name.
    
- Focus on roles like:
    
    - CEO, CTO, CISO
        
    - Developers
        
    - HR (often more vulnerable to social engineering)
        
    - IT / Helpdesk (may share technical tools and workflows)
        

> 🔎 Tip: Use **Google Dorking**

```bash
site:linkedin.com/in "Company Name" AND ("Engineer" OR "Admin")
```

#### 📄 **Details to Collect**:

- Full names
    
- Job roles and hierarchy
    
- Work emails (or patterns, e.g., [firstname.lastname@company.com](mailto:firstname.lastname@company.com))
    
- Personal projects or tech stacks used
    
- Phone numbers
    
- CVs/resumes
    
- Opinions or posts (useful for pretexting in social engineering)
    

---

### 🛠 3. Tools to Use

#### 📥 theHarvester:

- Gathers **emails**, **subdomains**, and **employee names** from public sources (Google, Bing, LinkedIn, etc.)
    
- Example:
    
    ```bash
    theHarvester -d target.com -l 500 -b google
    ```
    

#### 🕵️‍♂️ Infoga:

- Python tool focused on **email enumeration** and **leak checks**.
    
- It can tell if an email has **appeared in known data breaches**.
    
- Example:
    
    ```bash
    python infoga.py --domain target.com --source all
    ```
    

#### 🌍 PeopleFinder / PeekYou / Pipl (OSINT People Search Engines):

- Use to find:
    
    - Names
        
    - Addresses
        
    - Phone numbers
        
    - Social media profiles
        
    - Public records
        

> ⚠️ Always use **legal OSINT** tools and avoid violating privacy policies.

---

### 🧠 4. Why Is This Important?

- **Email-based attacks** (like spear phishing) often rely on knowing **real employee names** and roles.
    
- Publicly exposed employee data might be found in **data breaches**, resumes, or tech blogs.
    
- If an employee's email has been leaked before, it might be a **way in** through reused passwords or forgotten access.
    

> 💡 Example: If a CEO’s email is found in a **LinkedIn data leak**, that’s a high-value target for account takeover.
